# Security Policy

Please report vulnerabilities privately.

Do not open public issues for vulnerabilities, secrets, private endpoints,
operator logs, or hosted-service incidents. Use GitHub private vulnerability
reporting when available, or contact the maintainer through the security contact
listed on the TraceDB website.

TraceDB public repositories cover the downloadable database distribution,
protocol, SDKs, examples, and RFCs. Hosted TraceDB service operations are
handled separately.
